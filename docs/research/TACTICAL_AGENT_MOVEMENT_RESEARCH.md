# Research Request: Tactical Agent Movement for SCOUT C2 3D View

## Context

SCOUT C2 currently shows 5 agents moving on a procedural village map. Movement is generated by A* pathfinding between building waypoints, then linearly interpolated into small steps. Agents move at constant speed, stop instantly, and lack tactical behavior. The visual result feels like dots sliding on a map, not like a special-ops team clearing a village.

## Goal

Make agent movement look like a real tactical squad in a Rainbow Six Siege-style "god camera" view: believable, dynamic, readable, and cinematic.

## Research Questions

1. What movement behaviors make AI agents look tactical in games like Rainbow Six Siege, Door Kickers, Zero Hour, or SWAT 4?
   - Entry stacking / bounding overwatch
   - Slice-the-pie cornering
   - Door breaching pauses
   - Cover-based movement (pause at corners, hug walls)
   - Hand signals / posture changes
   - Speed variations (walk → sprint → crouch)

2. What are simple algorithms to implement these behaviors without full animation rigs?
   - Steering behaviors: arrive, seek, wall following, cover seeking
   - Waypoint action nodes: "pause 2s", "face door", "cover left"
   - Formation movement (diamond, wedge, line)
   - Randomized human-like jitter (not robotic straight lines)

3. How do tactical shooters visualize intent and state?
   - Planned path lines (dashed ghost lines)
   - Icons for stance: standing, crouched, sprinting
   - Breach markers, door interaction icons
   - Communication lines between agents
   - Slow-motion emphasis on key moments (breach, contact)

4. What camera work makes the view cinematic?
   - Smooth follow cam with lookahead
   - Cinematic orbit during idle
   - Snap-to-action when alert triggers
   - Multiple camera bookmarks (drop cam, breach cam, overview cam)

5. What open-source resources exist?
   - Three.js examples for path visualization
   - Recast-Navigation / NavMesh for obstacle-aware movement
   - Tactical shooter GDC talks on AI
   - Military doctrine sources for small-unit tactics (FM 3-21.8, MOUT)

## Deliverables

- Shortlist of 3-5 implementable tactical behaviors ranked by visual impact / effort.
- Pseudocode or code snippets for the top 2 behaviors.
- Visual reference list (screenshots, game names, specific mechanics).
- Recommendation for camera system upgrades.

## Constraints

- Must work in browser with Three.js (no external animation assets required).
- Must run at 60 FPS with 5 agents and ~90K point cloud.
- Should not require changes to the simulator coordinate system.
- Prefer procedural / code-generated motion over GLB animations.
